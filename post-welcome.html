On this page

왜 새 공간을 만들었나 어떻게 사용할 것인가 다음 단계

이 페이지는 새 개발 블로그의 디자인과 게시 흐름을 확인하기 위한 첫 기록입니다.

왜 새 공간을 만들었나

기존 Tistory 기록을 이어가면서도, 코드와 콘텐츠를 직접 관리할 수 있는 공간이 필요했습니다. GitHub Pages 위에 가볍고 빠른 정적 사이트를 두고 작은 기록부터 다시 쌓아갈 예정입니다.

어떻게 사용할 것인가

개발 중 만난 문제를 해결한 과정, 새로 배운 개념, 다시 찾을 가능성이 높은 명령과 설정을 중심으로 적습니다. CS와 AI처럼 긴 설명이 필요한 글은 목차와 참고 링크를 붙이고, 게임 개발과 그래픽스 글은 코드·로그·다이어그램을 함께 둘 생각입니다. 

const workflow = [
  "read",
  "build",
  "debug",
  "explain"
];
기록은 완벽한 설명보다, 다음 탐색을 시작할 수 있는 좌표에 가깝다.

다음 단계

사이트 구조가 안정되면 기존 글을 필요한 만큼 옮기고, 태그와 검색 기능도 콘텐츠에 맞춰 다듬습니다. 우선은 글을 오래 읽기 좋은 본문 폭과, 나중에 찾기 쉬운 아카이브 구조를 먼저 맞춰 둡니다.
